API REST para WhatsApp

WUZAPI é uma implementação da biblioteca whatsmeow como um serviço de API RESTful simples, com suporte para múltiplos dispositivos e sessões simultâneas.

Recursos Principais

WUZAPI oferece uma API completa para interagir com o WhatsApp de forma eficiente sem usar recursos pesados como Puppeteer ou emuladores Android.

Performance Alta

Comunicação direta com os servidores do WhatsApp via websocket, resultando em menor uso de memória e CPU.

Múltiplas Sessões

Suporte para múltiplos dispositivos e usuários simultâneos na mesma instância.

Mensagens Rich

Webhooks

Como Usar

Instruções para começar a usar a WUZAPI rapidamente.

Pré-requisitos

Para executar a WUZAPI, você precisa:

  • Go ( Linguagem de Programação Go)
  • Docker (opcional, para contêinerização)
  • Postgres (opcional, pode funcionar independentemente usando SQLite)

Compilação

Compile o projeto com o seguinte comando:

go build .

Configuração

A WUZAPI usa um arquivo .env para configuração. Aqui estão as configurações necessárias:

Para PostgresSQL

WUZAPI_ADMIN_TOKEN=your_admin_token_here
USER=wuzapi
PASSWORD=wuzapi
NAME=wuzapi
HOST=localhost
PORT=5432
America/New_York

Para SQLite

WUZAPI_ADMIN_TOKEN=your_admin_token_here
TZ=America/New_York

Execução

Por padrão, o serviço será iniciado na porta 8080. Você pode alterar o comportamento com os seguintes parâmetros:

  • -address: define o endereço IP para vincular o servidor (padrão 0.0.0.0)
  • -port: define o número da porta (padrão 8080)
  • -logtype: formato para logs, console (padrão) ou json
  • -wadebug: habilita debug do whatsmeow, níveis INFO ou DEBUG são suportados
  • -sslcertificate: Arquivo de Certificado SSL
  • -sslprivatekey: Arquivo de Chave Privada SSL
  • -skipmedia: Não faça download automático de mídia de mensagens recebidas
  • -osname: Nome do Sistema Operacional para conexão em WhatsApp (padrão "Mac OS 10")
  • -admintoken: Token de Segurança para autorizar ações administrativas (listar/criar/remover usuários)
Exemplo:
./wuzapi -logtype json

Criando Usuários

Para abrir sessões, você primeiro precisa criar um usuário e definir um token de autenticação para ele. Você pode fazer esse login no Painel, ou usando a API diretamente:

curl -X POST http://localhost:8080/admin/users \
-H "Authorization: $WUZAPI_ADMIN_TOKEN" \
-H "Content-Type: application/json" \
-d '{"name": "John", "token": "Z1234ABCCXD"}'

Uma vez que os usuários são criados, você pode se comunicar com a API passando o cabeçalho Token como um método de autenticação simples. Você pode ter vários usuários (números diferentes) no mesmo servidor. Para cada usuário você precisará Conectar ao WhatsApp e então escanear um QR Code ou Parear via número de telefone.

Recursos Web

O daemon também serve alguns arquivos estáticos web úteis para desenvolvimento/teste que você pode carregar com seu navegador:

  • Referência da API Swagger em /api
  • Uma página web de exemplo para conectar e escanear QR codes em /login (onde você precisará passar ?token=1234ABCD)
  • Um painel de gerenciamento em /dashboard
